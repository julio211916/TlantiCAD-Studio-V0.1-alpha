# design-systems Rules

## Purpose

Design system lookup, CSS best practices, and AI-driven UI component generation via MCP tools.

## Best Practices

- Never hardcode hex colors — use design tokens
- Never reinvent existing components
- Never use px for spacing — use rem/em
- Use semantic search to find patterns

- design_systems\_\_get_tokens
- design_systems\_\_get_patterns
