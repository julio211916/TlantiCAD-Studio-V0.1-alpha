# doc-coauthoring Rules

## Purpose

Collaborative document creation via structured three-stage workflow.

## Best Practices

- Ask clarifying questions before writing
- Brainstorm 5-20 options per section
- Use str_replace for targeted edits
- Test with reader sub-agents
- Scaffold full document early with placeholders

## Three Stages

1. Context Gathering: Clarify type, audience, constraints
2. Section Refinement: Brainstorm, draft, refine, confirm
3. Reader Testing: Spawn reader sub-agent to test clarity
