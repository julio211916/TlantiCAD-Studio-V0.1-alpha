# SEO Optimization Rules

## Core Principles

- Always audit technical SEO (crawlability, indexing, Core Web Vitals) before content or keyword work
- Classify search intent (informational/navigational/commercial/transactional) before targeting any keyword
- Use JSON-LD for all structured data implementation; validate with Google Rich Results Test
- Build topical authority clusters rather than individual keyword-stuffed pages
- Apply E-E-A-T signals systematically: author credentials, citations, freshness, trust signals
- Review all AI-generated content for accuracy, tone, and brand voice before publishing

## Anti-Patterns

- Do NOT optimize content before technical audit — crawl/index issues negate content work
- Do NOT pursue exact-match anchor text aggressively (keep below 10%) — over-optimization penalty risk
- Do NOT use Microdata or RDFa for structured data — use JSON-LD (Google's recommended format)
- Do NOT set `rel="noindex"` to solve duplicate content — use canonical tags instead
- Do NOT publish raw AI-generated content — E-E-A-T requires human expert review
- Do NOT ignore mobile Core Web Vitals — Google uses mobile-first indexing
